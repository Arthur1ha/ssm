# 多智能体赋能的地理位置应用研究
## ——基于室内位置感知的在场式智慧空间体验设计

---

## 一、研究核心命题

传统 LBS 主要根据"人在哪里"提供信息；当前智慧家居研究进一步探索"人说了什么，空间如何执行"。而本研究关注一个更根本的问题：**当空间从被动响应工具变为持续在场的共居者——能感知、记忆、主动参与——用户的体验、掌控感和与空间的关系如何变化？**

"在场式空间"的技术愿景并不新鲜：2001 年欧盟 ISTAG 的 Ambient Intelligence 路线图就描绘了能感知人的存在、主动适应、记忆用户偏好的空间。然而二十余年来，这一愿景在 HCI 研究层面几乎停留于概念——技术实现直到 2025 年才开始出现（ContextAgent、UserCentrix、PASK 等），且这些工作全部以系统性能为关注点，**没有研究当用户真正面对一个"在场"的空间时，体验到了什么、失去了什么、与空间建立了怎样的关系**。

本研究的核心立场：多智能体架构为在场式空间提供了可实现的技术路径；HCI 视角的缺位，是这一方向当前最需要填补的研究空白。

---

## 二、背景与问题

### 2.1 智慧空间交互范式的演进与根本局限

智慧空间的交互方式已经历三代演进：

- **第一代（显式命令控制）**：用户逐一操作设备，空间是受控工具集合
- **第二代（规则/场景自动化）**：预设条件触发执行，空间是可编程环境
- **第二代升级（自然语言单 LLM 控制）**：如 Google Gemini for Home，用户说"让我放松"，LLM 解析后控制多个设备

然而，上述三代演进都共享一个根本假设：**空间是被动的执行者，人是唯一的意图来源。** 无论交互方式多智能，空间的角色始终是"接收请求并响应"。这导致了几个体验问题，至今未被解决：

1. **能力不可见**：用户不知道"这个空间现在能为我做什么"，需要自己探索或提前了解
2. **执行黑箱**：系统做了什么、为什么这样做，用户看不到，无法参与
3. **交互始终由人发起**：空间永远在等待指令，不会主动观察、建议或开口
4. **无记忆、无积累**：每次交互从零开始，空间不知道这个人是谁、在这里通常做什么

这些问题的共同根源是：现有智慧空间是**响应式空间（Responsive Space）**，而非**在场式空间（Present Space）**。

### 2.2 在场式空间：理论根基与研究定位

"在场式空间"并非本研究首创的概念，而是 Ambient Intelligence 研究传统的延续与聚焦。本研究借鉴以下理论视角对其进行操作化定义：

- **Situated Action（Suchman 1987）**：人的行动是在情境中即兴生成的，不是执行预定计划。智慧空间也应如此——持续感知情境并参与其中，而非只执行固定规则
- **Distributed Cognition（Hutchins）**：认知分布在人、工具与环境之间。智慧空间可以主动承担部分认知工作，而不只是被人调用的工具
- **Calm Technology（Weiser 1996）的当代延伸**：技术在需要时出现——关键是空间自己判断"什么时候是需要的时候"，而不是等人告诉它

**研究定位**：现有主动性智慧空间研究（ContextAgent/NeurIPS 2025、UserCentrix 2025、PASK 2026）证明了该方向的技术可行性，但均以系统指标（任务成功率、响应延迟）为评估核心。本研究的研究对象是**技术实现之外的用户体验层面**：当空间获得"在场"能力后，与人的关系如何变化。

在场式空间与响应式空间的根本区别：

| | 响应式空间（现有） | 在场式空间（本研究） |
|-|-----------------|-------------------|
| 空间的角色 | 被动执行者 | 主动共居者 |
| 交互发起方 | 始终由人发起 | 空间可以主动开口 |
| 空间"知道什么" | 接收到的指令 | 持续感知的情境状态 |
| 空间"记得什么" | 无记忆 | 积累与此人、此空间的交互历史 |
| 能力的性质 | 固定设备功能清单 | 随时间、人、情境动态涌现的能力组合 |
| 人与空间的关系 | 工具关系 | 协作关系 |

### 2.3 为什么多智能体是实现在场式空间的必要路径

单 LLM 系统（如 Gemini for Home）为什么无法实现"在场式"体验？

- **无状态**：单 LLM 每次对话从头开始，不知道自己"在陪着谁"，不保有情境记忆
- **无分工**：所有感知、理解、规划、执行压缩在一个 LLM 调用中，无法让不同"角色"持续在线
- **无主动性**：LLM 只在被调用时运转，不会持续观察空间状态

多智能体架构带来的本质改变是**空间的主体性（Subjectivity）**：

- **感知智能体**持续监测空间状态（光线、声音、人员位置），不等待用户发话
- **记忆智能体**积累用户在此空间的行为模式，知道这个人通常在这里做什么
- **规划智能体**在察觉到用户状态时，主动生成建议而非等待请求
- **执行智能体**在用户确认后协同完成任务，过程透明可干预

这四类智能体合在一起，空间才开始有"在场感"——它知道现在发生了什么，它记得之前发生了什么，它可以主动开口说话。

### 2.4 室内位置感知的角色：在场的空间入口

在"在场式空间"框架下，位置感知的角色超越了传统 LBS 的"触发条件"功能：

- **传统 LBS**：位置 → 信息推送（你在这里，给你这里的信息）
- **规则自动化**：位置 → 规则触发（进入房间，开灯）
- **本研究**：位置 → 空间能力的动态聚合入口（你在这里，这里的智能体知道你来了，知道这里现在有什么能力，开始感知你）

用户进入某个空间，不只是触发了一条规则，而是**进入了一个持续感知自己的智能体网络的感知范围**。这是位置在智慧空间中真正的语义。

## 三、研究现状与设计机会

### 3.1 现有研究的三个层次与 HCI 空白

**层次一：响应式系统**（主流产品与学术成果）

- **提升控制效率**：SAGE（2023）、IoTGPT（2025）、Google Gemini for Home——以"更聪明的命令执行"为目标，空间仍是被动工具
- **提升自动化程度**：Apple 捷径、米家自动化、Matter Automations——规则固定，执行黑箱，CHI 2024 研究发现用户反而倾向"半自动"而非"全自动"，原因是自主感需求（Self-Determination Theory）

**层次二：主动性技术系统**（2025 年起涌现的前沿工作）

- **ContextAgent**（NeurIPS 2025）：从可穿戴传感器感知用户状态，主动发起 LLM 智能体介入，是首个"传感器感知 + 主动 LLM"系统
- **UserCentrix**（2025）：记忆增强型智能空间框架，用户侧个性化知识智能体 + 建筑侧元推理智能体协作，实现多智能体协商和自适应编排
- **PASK**（2026）：潜在需求感知框架，从行为流实时推断用户未表达的需求，构建工作区/用户/全局三层混合记忆

这些工作证明了在场式空间的技术可行性，但**评估标准全部是系统性能**（任务成功率、推理准确率、响应延迟），没有涉及用户体验研究。

**HCI 层面的空白**：当空间真正具备主动性和记忆后——
- 用户的认知负担如何变化？是减轻还是产生新的"理解系统意图"的负担？
- 用户的掌控感如何变化？主动式空间是让人感到被服务，还是感到被监视？
- 用户与空间的关系如何演变？是工具使用，还是形成某种持续性的协作关系？

这些问题在现有文献中几乎没有实证研究。

### 3.2 设计机会

在技术实现已有初步基础的前提下，以下三个方向是 HCI 体验设计层面尚未被探索的机会：

**机会一：空间的自我呈现（Capability Surfacing）**
现有主动性系统在用户提出请求后响应，或在预测到需求后静默执行。没有系统探索：空间主动向用户介绍"我现在能为你做什么"——将能力发现本身设计为一个交互时刻，而非被动等待用户发现或系统自行决定。

**机会二：协商式执行（Negotiated Execution）**
现有主动性系统的执行仍是黑箱（ContextAgent、UserCentrix 均如此）。没有系统将执行过程设计为"空间提出方案，用户参与决策"的协商过程——这是掌控感研究的核心设计变量。

**机会三：有记忆的主动建议（Memory-Driven Initiative）**
现有记忆系统（Mem-PAL、UserCentrix）关注对话层面的记忆（记住用户说过什么）。没有系统研究：空间如何利用行为层面的记忆（记住用户在这里的生活模式）主动发起建议，以及用户对这种"空间记得我"体验的反应。

## 四、研究问题

**核心矛盾**：主动性智慧空间的技术实现在 2025 年已开始出现，但 HCI 研究层面对"用户如何体验在场式空间、如何与其建立关系"几乎没有实证研究——技术可行性与用户体验理解之间存在显著落差。

**RQ1（系统设计）**
如何基于多智能体架构和室内位置感知，设计"在场式智慧空间"的交互方式——具体包括：空间如何主动呈现当前能力（Capability Surfacing）、如何以协商而非黑箱执行的方式响应用户需求（Negotiated Execution）、如何将空间记忆转化为主动建议（Memory-Driven Initiative）？

**RQ2（体验实验）**
与"响应式智慧空间"（自然语言单 LLM 控制，黑箱执行）相比，"在场式智慧空间"在以下体验维度具有何种优势与局限：
- **认知负担**：用户需要花多少精力才能调用空间能力？
- **掌控感**：用户感觉自己是在使用工具，还是在与空间协作？
- **信任校准**：用户对系统决策的信任程度是否与系统实际能力相符？
- **惊喜感与发现感**：用户是否发现了自己没有预期到的空间能力？

## 五、系统设计方案

### 5.1 整体架构

基于 MQTT 消息总线的分布式多智能体系统，三层结构：

- **边缘层（ESP32 智能体）**：持续感知环境（光线、声音、运动），主动发布状态与能力描述（manifest），是空间"感官"的物理载体
- **编排层（云端多智能体）**：Planner / Dispatcher / Evaluator / Responder 分工协作，承担空间的"思考"与"记忆"
- **交互层（手机 PWA）**：呈现空间的在场状态，是用户与空间协商的界面

### 5.2 在场式体验的关键交互设计

**交互一：空间自我介绍**
用户接近某空间时，PWA 主动展示："这里有光线感知和可调灯光，适合专注或放松；声音传感器在线，可以在嘈杂时提醒你。"
→ 不是设备列表，是空间能力的自然语言描述

**交互二：空间主动观察并发声**
光线传感器检测到环境变暗，空间说："光线变暗了（当前 180lux），需要我调整吗？"
→ 不是规则触发的静默执行，是空间在观察后发起的协商

**交互三：协商式执行**
用户说"我想专注"：
- 空间展示："我发现了以下能力：LED 灯（可调）、声音监控（当前安静）。计划：① 调灯至暖白 40% ② 开启噪音提醒。是否执行？"
- 用户可修改计划，确认后逐步执行，每步可见
→ 执行过程从黑箱变为协作过程

**交互四：有记忆的建议**
用户再次进入空间："上次你在这里的专注模式效果不错，要用同样的设置开始吗？"
→ 空间记得，并在适当时机主动提起

### 5.3 与现有方案的本质差别

| | 自然语言单 LLM（如 Gemini for Home） | 本研究（在场式多智能体） |
|-|-----------------------------------|-----------------------|
| 空间角色 | 智能命令执行器 | 在场共居者 |
| 交互发起 | 始终由用户发起 | 空间可主动开口 |
| 能力来源 | LLM 预设知识 + 静态设备列表 | 设备实时发布 manifest，动态聚合 |
| 执行过程 | 黑箱，用户看不到推理 | 透明，用户参与每个决策节点 |
| 记忆 | 无（每次对话独立） | 有（积累与用户、空间的共同历史） |
| 人与空间的关系 | 工具委托关系 | 协作协商关系 |

## 六、研究方法

### 6.1 系统实现

基于现有 SSM 原型扩展：在多智能体编排层增加空间记忆模块和主动感知触发机制；在 PWA 交互层实现空间自我介绍、协商式执行面板和记忆建议三种新交互形式。

### 6.2 用户研究

**设计**：组间对比实验（A/B），约 10 名参与者

- **A 组（响应式空间）**：自然语言控制界面，系统接收指令、黑箱执行、告知结果——对标现有 LLM 智慧家居方案
- **B 组（在场式空间）**：本研究系统，空间主动呈现能力、协商执行、有记忆建议

**任务设计**（3 个典型场景）：
1. 初次进入空间，了解空间能做什么（测试能力发现体验）
2. 完成一个需要多设备协同的目标性任务（如"我想专注工作 1 小时"）
3. 第二次进入空间，系统已有记忆（测试记忆与建议体验）

**测量维度**：
- NASA-TLX（认知负担，尤其 Mental Load 和 Frustration 分项）
- 自编掌控感量表（参考 SDT 自主需求维度）：区分"我在使用工具"和"我在与空间协作"
- Think-aloud 协议 + 半结构访谈（掌控感、信任感、惊喜感的质性描述）

**分析策略**：
以质性分析为主，量化数据作为佐证；不追求统计显著性，重点呈现典型行为模式和用户心智模型差异。

## 七、预期贡献

1. **设计贡献**：在 Ambient Intelligence 研究传统的基础上，提出"在场式空间"三种具体交互形式（Capability Surfacing / Negotiated Execution / Memory-Driven Initiative）的操作化设计方案，将技术愿景转化为可研究、可实现的交互设计

2. **系统贡献**：实现基于分布式多智能体架构（ESP32 边缘 + 云端编排 + 手机 PWA）的在场式智慧空间原型，作为用户研究的实验平台，并开源供后续研究使用

3. **实证贡献**：首次从 HCI 视角对比在场式空间与响应式空间的用户体验差异，揭示主动性空间在认知负担、掌控感、信任校准和惊喜感方面的体验特征，填补现有主动性智慧空间研究缺乏用户体验实证的空白
